# iCloud CLI Agent Guide

This directory contains `icloud-pp-cli`, a hand-written CLI for querying iCloud data stored locally on macOS. It was not generated by CLI Printing Press — local edits are the canonical version. Treat all source files here as the authoritative copy.

## Local Operating Contract

Always run doctor before any other command to confirm macOS, Photos.app, and library access:

```bash
icloud-pp-cli doctor
```

Use `--help` to discover current commands and flags:

```bash
icloud-pp-cli --help
icloud-pp-cli photos --help
icloud-pp-cli photos top --help
```

Add `--agent` to any command for JSON output, compact fields, no color, and non-interactive defaults:

```bash
icloud-pp-cli photos top --agent
icloud-pp-cli photos storage --agent
icloud-pp-cli photos stats --agent
```

## Platform Constraint

This CLI is macOS-only. Every database command guards against non-Darwin platforms and returns exit code 10 with a clear message. Do not attempt to run on Linux or Windows.

## Read vs. Write

The `photos` subcommands fall into two categories:

**Read-only** (`stats`, `storage`, `top`, `videos`) — open Photos.sqlite with `mode=ro&_query_only=1`. Safe to run at any time; no Photos.app involvement.

**Write** (`delete`, `download`) — drive Photos.app via AppleScript (`osascript`):
- `photos delete` requires `--confirm` to move items to Recently Deleted. Items are NOT permanently deleted — they stay in Recently Deleted for 30 days.
- `photos download` exports originals to a local folder. When `--sensitive` is used (targets Apple's on-device nudity-flagged assets), `--confirm` is also required to prevent accidental bulk export of private content.

## Local Customizations

This CLI is hand-written (not generated), so all code is intentional. The `.printing-press-patches.json` at this directory's root documents the three design decisions that differ from a typical generated CLI:

1. `macos-only-guard` — structured error on non-Darwin
2. `applescript-delete` — Photos.app scripting for deletion
3. `doctor-sectioned-output` — System / Library / Assets pre-flight sections

`grep -rn 'PATCH' .` surfaces any additional inline customization comments added after initial authoring.
